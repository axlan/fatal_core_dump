Using GDB

GDB is a complex and versatile program, so here I'm going to focus on the commands that are most applicable for analyzing this core dump.

  • backtrace - Print information about the call stack
  • frame - Select the stack frame to examine. This lets you examine variables from caller functions
  • info symbol - Print info for any symbols that map near a memory address
  • print - Print variable or data
  • examine memory - Print out raw binary data at memory locations
  • disassemble - Print memory as assembly instructions. For example, disassemble $pc-64,$pc+64 will print the assembly for the instructions within 64 bytes of the current instruction pointer
